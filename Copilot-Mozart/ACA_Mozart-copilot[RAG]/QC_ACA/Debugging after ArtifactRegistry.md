deploy-rag:

**ข้อเสีย:**
- Services อาจ out of sync (version ต่างกัน)

---

### **Option B: Deploy ทั้งหมดหรือไม่เลย (All-or-Nothing)** ⭐ แนะนำ
```yaml
deploy-all:
  needs: [build-gateway, build-frontend, build-mcp-core, build-rag]
  if: success()  # ทุกตัวต้องผ่าน
  # ถ้าตัวใดตัวหนึ่ง fail → ไม่ deploy เลย
```

**ข้อดี:**
- Services sync กันเสมอ
- ไม่มี version mismatch

**ข้อเสีย:**
- ถ้า 1 ตัว fail = ทุกตัวไม่ได้ deploy

---

### **Option C: Matrix Strategy (Pro Level)** ⭐⭐ มืออาชีพใช้
```yaml
jobs:
  build:
    strategy:
      fail-fast: false  # ถ้าตัวนึง fail ตัวอื่นยังรันต่อ
      matrix:
        service: [frontend, gateway, mcp-core, rag]
    steps:
      - run: build ${{ matrix.service }}
  
  deploy:
    needs: [build]
    if: always()  # รันเสมอ
    strategy:
      matrix:
        service: [frontend, gateway, mcp-core, rag]
    steps:
      - if: needs.build.result == 'success' || contains(needs.build.outputs.successful, matrix.service)
        run: deploy ${{ matrix.service }}
```
