### 4.1 RAG (ACA_Mozart – Spec Engine)

- RAG **ไม่ยิง DB ตรง**
    
- ใช้งานผ่านเอกสารใน `rag_knowledge/db/` ที่อธิบาย:
    
    - โครงสร้างตาราง + kind
        
    - ตัวอย่าง JSON ต่อ kind
        
    - วิธี map จาก `ProjectInputSpec` → code จริงใน catalog
        
- เอกสารเหล่านี้ถูก index ใน `knowledge_index.json` ใน group `catalog_schema`
    
- เวลา RAG สร้าง `ProjectInputSpec` จะ:
    
    - เลือก `template_code`, `device_code`, `rule_profile_id` จาก code ที่ระบุในเอกสาร
        
    - ปล่อยให้ MCP เป็นคนไป query view ตาม code เหล่านี้
        

### 4.2 MCP Core v2.0

- ใช้ DAL / client ฝั่ง Python เชื่อม Supabase / Postgres
    
- ดึงข้อมูลจาก view ตาม kind ที่ต้องใช้ แล้วค่อยคำนวณด้วย pandapower / engine อื่น
    
- ถือว่าค่าใน catalog เป็น **แหล่งความจริงสุดท้าย**  
    ถ้าจะ upgrade ตาราง → ทำผ่าน migration + seed ใหม่
