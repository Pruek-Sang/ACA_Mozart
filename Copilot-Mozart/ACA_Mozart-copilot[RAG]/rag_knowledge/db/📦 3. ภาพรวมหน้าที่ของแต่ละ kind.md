### 3.1 COMPONENT

- เก็บข้อมูลชิ้นอุปกรณ์ เช่น เต้ารับ, สวิตช์, ดวงโคม
    
- ฟิลด์หลัก: `component_type`, `voltage_rating_v`, `current_rating_a`, `ip_rating`, `mount_height_mm`, `brand`, `model`, ฯลฯ
    
- ใช้โดย:
    
    - MCP / Layout: เลือกอุปกรณ์จริง, สร้าง BOM, เช็ก IP rating ฯลฯ
        
    - RAG: ใช้ผ่านเอกสารตัวอย่าง / contract เท่านั้น
        

### 3.2 CABLE_SPEC

- เก็บสเปกสายไฟที่ใช้คำนวณโหลด, VD, fault, ราคาก่อสร้าง
    
- ฟิลด์สำคัญ: `size_mm2`, `ampacity_in_conduit_a`, `resistance_ohm_per_km_20c`, `reactance_ohm_per_km`, `price_thb_per_m`, `standard_reference`
    
- ใช้โดย:
    
    - MCP Core v2.0 + pandapower สำหรับเลือกสายและคิด VD
        
    - RAG: อ้างอิงผ่านเอกสาร (เช่น “สาย THW 2.5 mm² รองรับกี่แอมป์”)
        

### 3.3 CIRCUIT_TEMPLATE

- เทมเพลตวงจร เช่น วงจรเต้ารับทั่วไป 20A
    
- ฟิลด์สำคัญ: `circuit_type`, `breaker_rating_a`, `wire_size_mm2`, `max_outlets`,  
    `dedicated_circuit`, ฯลฯ
    
- ใช้โดย:
    
    - MCP: map วงจรให้ห้อง/โหลด
        
    - VALIDATION_RULE: ใช้ข้อมูลนี้ในการเช็กโหลดรวมวงจร
        

### 3.4 ROOM_TEMPLATE

- เก็บรูปแบบห้อง (living_room, bedroom, kitchen ฯลฯ)
    
- มี base load, จำนวนเต้ารับขั้นต่ำ, typical_appliances, compliance limit
    
- ใช้โดย:
    
    - RAG ตอนสร้าง `ProjectInputSpec` (รู้ว่า room_type ไหนควรมีอะไร)
        
    - MCP ตอนกำหนดจำนวนวงจร/เต้ารับเริ่มต้น
        

### 3.5 APPLIANCE

- รายการอุปกรณ์ไฟฟ้าตามหมวด เช่น พัดลม, ตู้เย็น, ไมโครเวฟ
    
- ฟิลด์สำคัญ:
    
    - `power_w`, `running_current_a`, `startup_current_a`
        
    - `requires_dedicated_circuit`
        
    - `typical_rooms`, `usage_hours_per_day`
        
    - `brand`, `model`, `price_thb`
        
- ใช้โดย:
    
    - MCP: คิดโหลดจริงต่อวงจร
        
    - RAG: ใช้อ่านเท่านั้น เพื่อแปลง requirement เป็น spec
        

### 3.6 PLACEMENT_RULE / VALIDATION_RULE / DERATING_FACTOR / อื่น ๆ

- PLACEMENT_RULE: กติกาการวางอุปกรณ์ในแปลน (ระยะห่าง, zone ที่ห้ามวาง)
    
- VALIDATION_RULE: ตรวจแบบหลังจากวาง (โหลดรวมวงจร, IP, VD ฯลฯ)
    
- DERATING_FACTOR: ตาราง derating ตามอุณหภูมิ ฯลฯ
    
- GEOMETRY_FILTER / QA_PLAN / ZONE_BUNDLE / PANELBOARD / ROUTING_RULE / PROJECT_CONFIG:
    
    - ใช้สำหรับ layout engine, QA, grouping โปรเจกต์ และ config ทั่วไป
        
    - รายละเอียดโครง JSON ดูจาก seed และ `CATALOG_CONTRACT.md`
