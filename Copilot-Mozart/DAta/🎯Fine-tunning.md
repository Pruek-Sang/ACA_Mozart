# 🎯 แนวทาง Fine-tuning RAG สำหรับความแม่นยำของ JSON

> สรุปโดย Aethelgard | วันที่ 28 พฤศจิกายน 2025

---

## ภาพรวม

ระบบ RAG ของเราทำหน้าที่แปลงคำพูดของมนุษย์ เช่น "แอร์ 12000 BTU" ให้กลายเป็นรหัสมาตรฐาน เช่น "AC-12000BTU" เพื่อส่งต่อให้ MCP Core ทำการคำนวณ

ปัญหาคือ คนพิมพ์หลายแบบมาก เช่น "แอร์ 1 ตัน" "เครื่องปรับอากาศ" "air 12000" ซึ่งทั้งหมดหมายถึงอย่างเดียวกัน แต่ LLM อาจตีความผิดได้

## สิ่งที่ต้องเตรียม

### 1. ตารางจับคู่อุปกรณ์

เตรียมรายการคำที่คนอาจพิมพ์ พร้อมรหัสที่ถูกต้อง เช่น

| คนพิมพ์ | รหัสที่ถูกต้อง |
|---------|---------------|
| แอร์ 9000 BTU | AC-9000BTU |
| แอร์ 1 ตัน | AC-12000BTU |
| เครื่องปรับอากาศ 12000 | AC-12000BTU |
| ไฟ LED | LIGHT-LED-10W |
| ดาวน์ไลท์ | LIGHT-LED-10W |
| พัดลมเพดาน | FAN-CEILING-60W |
| ทีวี | TV-200W |
| ปลั๊ก | SOCKET-16A |
| เตาแม่เหล็กไฟฟ้า | INDUCTION-3000W |
| ตู้เย็น | REFRIG-300W |
| เครื่องทำน้ำอุ่น | HEATER-3500W |

ยิ่งมีคำหลายแบบที่ชี้ไปรหัสเดียวกัน ระบบยิ่งฉลาด

### 2. ตารางจับคู่ห้อง

เตรียมกฎการเลือก template ตามลักษณะห้อง เช่น

| ลักษณะห้อง | รหัส template |
|-----------|--------------|
| ห้องนอนเล็กกว่า 12 ตร.ม. | ROOMT-BED-S |
| ห้องนอน 12-20 ตร.ม. | ROOMT-BED-M |
| ห้องนอนใหญ่กว่า 20 ตร.ม. | ROOMT-BED-L |
| ครัวทั่วไป | ROOMT-KITCHEN-LIGHT |
| ครัวไทย หรือ มีเตาไฟฟ้า | ROOMT-KITCHEN-HEAVY |
| ห้องน้ำมีเครื่องทำน้ำอุ่น | ROOMT-BATH-HEATER |

### 3. ตัวอย่างแบบเต็ม

เตรียมตัวอย่าง input-output คู่กัน 10-20 ชุด เช่น

**ตัวอย่างที่ 1: บ้าน 1 ชั้น 2 ห้องนอน**

ผู้ใช้บอกว่า:
- ห้องนอนใหญ่ 20 ตร.ม. มีแอร์ 12000 BTU
- ห้องนอนเล็ก 12 ตร.ม. มีพัดลมเพดาน
- ห้องนั่งเล่น 25 ตร.ม. มีแอร์ 18000 BTU กับทีวี
- ครัวมีตู้เย็นกับไมโครเวฟ
- ห้องน้ำมีเครื่องทำน้ำอุ่น

ระบบต้องสร้างออกมาได้ว่า:
- ห้องนอนใหญ่ ใช้ template ROOMT-BED-M เพราะ 20 ตร.ม.
- ห้องนอนเล็ก ใช้ template ROOMT-BED-S เพราะ 12 ตร.ม.
- ครัวใช้ ROOMT-KITCHEN-LIGHT เพราะไม่มีเตาไฟฟ้าหนัก
- ห้องน้ำใช้ ROOMT-BATH-HEATER เพราะมีเครื่องทำน้ำอุ่น
- แอร์ 12000 BTU แปลงเป็น AC-12000BTU
- พัดลมเพดาน แปลงเป็น FAN-CEILING-60W
- เป็นต้น

### 4. กรณีพิเศษ

เตรียมคำอธิบายสำหรับกรณีที่อาจสับสน เช่น

| กรณี | วิธีจัดการ |
|------|-----------|
| บอกว่า "แอร์" ไม่ระบุ BTU | ถามกลับ หรือ ใช้ 12000 BTU เป็น default |
| บอกว่า "แอร์ 1.5 ตัน" | แปลงเป็น 18000 BTU (1 ตัน = 12000 BTU) |
| บอกว่า "ครัวไทย" | ถือเป็นครัวหนัก ใช้ ROOMT-KITCHEN-HEAVY |
| บอกว่า "ห้องนอน master" | ถือเป็นห้องใหญ่ ใช้ ROOMT-BED-L |
| บอกว่า "ปลั๊กเครื่องซักผ้า" | ใช้ SOCKET-20A เพราะเป็นเครื่องใช้หนัก |

## วิธีนำไปใช้

### วิธีที่ 1: ใส่ตัวอย่างใน Prompt (ง่ายสุด)

ทุกครั้งที่เรียก LLM ให้ใส่ตัวอย่าง 2-3 ชุดไปใน prompt ด้วย เพื่อให้ LLM เห็นว่า "เออ เขาอยากได้แบบนี้" แล้ว LLM จะทำตาม

ข้อดี: ทำได้เลย ไม่ต้อง train อะไร
ข้อเสีย: ใส่ได้จำกัด เพราะ prompt มีขนาดจำกัด

### วิธีที่ 2: Fine-tune โมเดล

ใช้บริการ Vertex AI Tuning ของ Google ป้อนข้อมูลคู่ input-output เข้าไป 100+ ชุด แล้วให้ระบบ train โมเดลใหม่ที่เข้าใจงานของเราโดยเฉพาะ

ข้อดี: แม่นมาก เพราะโมเดลถูกสอนมาเพื่องานนี้
ข้อเสีย: ต้องเตรียมข้อมูลเยอะ ใช้เวลา ใช้เงิน

### วิธีที่ 3: Hybrid (แนะนำ)

ใช้ตารางจับคู่เป็นตัวช่วยก่อน ถ้าคำที่ผู้ใช้พิมพ์ตรงกับในตาราง ก็ใช้รหัสจากตารางเลย ไม่ต้องให้ LLM คิด

ถ้าไม่ตรง ค่อยให้ LLM คิด

ข้อดี: ประหยัด ลดการเรียก LLM ลดโอกาสผิดพลาด
ข้อเสีย: ต้องดูแลตารางให้ครบถ้วน
